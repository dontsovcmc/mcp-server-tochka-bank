# Разработка и диагностика

## Правила кода

### Безопасность

- **Пути для записи файлов** — валидировать через `_safe_output_path()`. Никогда не принимать произвольные пути от LLM без проверки. Функция кроссплатформенная (Windows/Linux/macOS):
  - Разрешены только домашняя директория и системный temp (`tempfile.gettempdir()`, `/tmp`).
  - Запись в dotfiles/dotdirs под home запрещена (`~/.ssh`, `~/.bashrc`, `~/.config/...`) — проверка через `os.sep + "." in relative_path`.
  - Пути разворачиваются через `os.path.realpath()` (защита от `..` и симлинков).
- **Парсинг JSON от пользователя** — всегда через `_parse_json(text, label)`, не через голый `json.loads()`. Пользователь должен получить понятное сообщение об ошибке, а не стектрейс `JSONDecodeError`.
- **Секреты** — только через переменные окружения (`os.getenv`). Не хардкодить токены, ИНН, номера счетов в коде или тестах. В тестах использовать вымышленные данные (ИНН `7700000000`, счёт `40702810100000000001`).
- **stdout зарезервирован** для JSON-RPC. Весь логгинг — только через `logging` в stderr.
- **HTTP-ответы в исключениях** — НИКОГДА не включать `resp.text` в `RuntimeError`. Тело ответа API может содержать персональные данные (ИНН, счета, имена), которые утекут в логи Claude, MCP-ответ или скриншоты. В исключение — только метод, путь и статус-код: `raise RuntimeError(f"GET {path} -> {resp.status_code}")`. Полный ответ логировать только на уровне `log.debug()`.

### Обработка ошибок

- **Никогда не глотать исключения молча.** `except Exception:` без логирования запрещён. Всегда логировать через `log.warning()` или `log.error()` с контекстом (что делали, для какого объекта).
- **HTTP-ошибки** — `tochka_api.py` бросает `RuntimeError` с методом, путём и статус-кодом (без тела ответа). Не перехватывать эти ошибки в tool-функциях — FastMCP сам вернёт ошибку клиенту.
- **Ошибки валидации** — бросать `RuntimeError` с понятным сообщением на русском. Пример: `raise RuntimeError(f"Товар '{name}' не найден")`.

### Стиль кода

- **Именование** — функции-хелперы с префиксом `_`. Имена должны быть читаемыми: `_to_json`, `_parse_json`, `_safe_output_path`, а не `_j`, `_p`, `_s`.
- **Дублирование** — каждая tool-функция самодостаточна (вызывает `_get_api()` + `_get_account()`). Это допустимое повторение, а не дублирование. Не выносить в декораторы/контекстные менеджеры.
- **Типы** — аннотировать параметры и возвращаемые значения. `dict`, `list`, `str` — не `Dict`, `List`, `Str` (используем встроенные типы Python 3.10+).
- **Импорты** — стандартная библиотека → сторонние пакеты → локальные модули, разделённые пустой строкой. Не импортировать неиспользуемые имена.
- **Модели** — Pydantic-модели в `models.py` только для тестов. Не дублировать модели с одинаковыми полями.
- **Линтер** — `ruff check src/ tests/` должен проходить без ошибок. Конфиг в `pyproject.toml`. Правила: E (стиль), F (ошибки), W (предупреждения), I (импорты). Максимальная длина строки — 120 символов.

### API-клиент (tochka_api.py)

- **Сессия** — `TochkaAPI` кэшируется в `server._api` (один экземпляр на процесс). Не создавать новый `TochkaAPI` на каждый вызов.
- **HTTP-хелперы** — `_get()`, `_post()`, `_delete()`, `_get_bytes()`. Для скачивания файлов использовать `_get_bytes()`, не писать ручные запросы.
- **Polling** — `get_statement_ready()` использует экспоненциальный backoff (2→4→8→16 сек), не фиксированный интервал. Для новых polling-функций — тот же паттерн.
- **Таймаут** — 30 секунд на все запросы. Для тяжёлых операций (скачивание PDF) может понадобиться увеличить.

### Тесты

- **Мокаем `TochkaAPI`** на уровне класса: `@patch("mcp_server_tochka_bank.server.TochkaAPI")`. Кэш API сбрасывается автоматически через фикстуру `_reset_api_cache` в `conftest.py`.
- **Пути в тестах** — использовать `os.path.realpath()` для сравнения путей (на macOS `/tmp` → `/private/tmp`). Temp-файлы создавать в `/tmp` явно: `NamedTemporaryFile(dir="/tmp")`.
- **Pydantic-валидация** — каждый тест валидирует mock-данные и ответ tool через модели из `models.py`.
- **Тестовые данные** — вымышленные ИНН, счета, имена. Никаких реальных персональных данных.

## Как Claude запускает MCP-сервер

Claude Code запускает MCP-сервер как **дочерний процесс** (subprocess) и общается с ним по **stdin/stdout** через JSON-RPC 2.0:

```
Claude Code                          MCP-сервер
    │                                    │
    │── spawn subprocess ──────────────►│  (command + args из конфига)
    │                                    │
    │── stdin: {"method":"initialize"} ─►│  handshake (таймаут 10 сек)
    │◄─ stdout: {"capabilities":...} ───│
    │                                    │
    │── stdin: {"method":"tools/list"} ─►│  получить список инструментов
    │◄─ stdout: [tools...] ─────────────│
    │                                    │
    │        ✓ Connected                 │
```

**Важно:**
- Сервер **не запускается в login shell** — `~/.bashrc`, `~/.zshrc` не выполняются
- **PATH может быть урезан** — команда может не находиться, хотя в обычном терминале работает
- **stdout зарезервирован** для протокола — любой `print()` в stdout ломает соединение (используйте stderr)

## `✗ Failed to connect` — что делать

**1. Запустить сервер вручную** — самый быстрый способ увидеть ошибку:

```bash
# Если ставили через uvx:
uvx mcp-server-tochka-bank

# Если ставили через pip:
mcp-server-tochka-bank
```

**2. Проверить что команда доступна:**

```bash
which mcp-server-tochka-bank   # или: which uvx
```

Если `not found` — пакет не установлен или не в PATH.

**3. Использовать абсолютный путь** (если PATH урезан):

```bash
# Узнать полный путь:
which mcp-server-tochka-bank
# Например: /Users/me/.local/bin/mcp-server-tochka-bank

# Использовать его в конфиге:
claude mcp remove tochka-bank
claude mcp add tochka-bank \
  -e TOCHKA_TOKEN=ваш_токен \
  -- /полный/путь/к/mcp-server-tochka-bank
```

**4. Посмотреть логи Claude Code:**

```bash
# macOS:
ls ~/Library/Logs/Claude/
tail -f ~/Library/Logs/Claude/mcp*.log

# Windows:
dir %APPDATA%\Claude\logs\

# Linux:
ls ~/.config/Claude/logs/
```

**5. Включить debug-режим:**

```bash
claude --mcp-debug          # отладка протокола MCP
claude --verbose             # подробный вывод
CLAUDE_DEBUG=1 claude        # максимум логов
```

## Частые проблемы

| Симптом | Причина | Решение |
|---------|---------|---------|
| `command not found` | Пакет не установлен или не в PATH | `pip install mcp-server-tochka-bank` или используйте абсолютный путь |
| `Failed to connect` без ошибки | Claude не может найти команду из-за урезанного PATH | Используйте абсолютный путь к команде |
| Сервер запускается вручную, но не в Claude | PATH в Claude отличается от PATH в терминале | Используйте абсолютный путь |
| `No module named mcp_server_tochka_bank` | pip установил в другой Python | Используйте entry point `mcp-server-tochka-bank` вместо `python -m` |
| Таймаут при подключении | Сервер не ответил за 10 секунд | Проверьте сеть и токен |
